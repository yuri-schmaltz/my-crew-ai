# ADR: Documentação Executável e Runbooks CrewAI

## Contexto
A documentação foi expandida com exemplos executáveis, troubleshooting, runbooks de release e métricas.

## Decisão
Manter docs sempre alinhadas ao código, priorizando exemplos que possam ser validados por novos usuários e scripts de automação.

## Consequências
- Onboarding facilitado e troubleshooting mais rápido.
- Redução de dúvidas recorrentes e erros operacionais.
- Base para automação de testes de documentação.
